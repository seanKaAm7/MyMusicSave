# MyMusicSave — 프로젝트 기획안

작성일: 2026-03-23

---

## 1. 프로젝트 개요

**MyMusicSave**는 Spotify 보관함을 시각화하고 분석해주는 개인 음악 통계 사이트다.
단순히 앨범 목록을 보여주는 것이 아니라, 사용자의 음악 취향을 다양한 방식으로 탐색하고
공유할 수 있는 공간을 제공한다.

기존 music-archive 사이트의 디자인 감성(다크 테마, Bebas Neue 타이포그래피, 미니멀)을 그대로 계승하되,
콘텐츠를 수동으로 입력하는 방식에서 벗어나 Spotify 데이터를 자동으로 동기화하는 구조로 전환한다.

---

## 2. 핵심 기능

### 2-1. Spotify 로그인 및 보관함 동기화
- Spotify OAuth 로그인 버튼
- 로그인 후 내 보관함(`/v1/me/albums`) 전체 자동 불러오기
- 커버 이미지, 트랙리스트, 발매일, 아티스트 정보 포함
- 보관함 업데이트 시 수동 새로고침으로 동기화

### 2-2. 보관함 시각화
- **장르 비율** — 파이차트 또는 바 그래프로 장르 분포 표시
- **연도별 분포** — 어느 시대 음악을 많이 저장했는지 타임라인
- **아티스트 통계** — 가장 많이 저장한 아티스트 순위
- **총 통계** — 총 앨범 수, 총 트랙 수, 총 재생시간 합산
- **최근 저장** — 최근에 보관함에 추가한 앨범 순서

### 2-3. 보관함 탐색
- 장르별 필터링
- 연도별 필터링
- 아티스트별 탐색
- 앨범 상세 페이지 (기존 music-archive 디자인 활용)

### 2-4. 공개 / 비공개 설정
- 기본값: 비공개
- 공개로 설정 시 고유 링크 생성 (`mymusicsave.netlify.app/u/username`)
- 링크를 통해 타인도 해당 유저의 보관함 통계 및 앨범 목록 열람 가능
- 공개 페이지에서는 Spotify 로그인 불필요

## 3. 기술 스택

| 역할 | 기술 | 이유 |
|------|------|------|
| 프론트엔드 | Vanilla JS (기존 유지) | 빌드 없음, 기존 코드 재활용 |
| 인증 | Spotify OAuth 2.0 | 보관함 접근 권한 필요 |
| 데이터베이스 | Supabase (PostgreSQL) | 무료, 인증 내장, 실시간 지원 |
| 서버 함수 | Netlify Functions | 기존 구조 유지, 토큰 교환 처리 |
| 차트 | Chart.js | 무료, 가볍고 커스터마이징 쉬움 |
| 배포 | Netlify | 기존과 동일 |

## 4. 데이터 구조 (Supabase)

### users 테이블
```
id            — Supabase 유저 ID
spotify_id    — Spotify 유저 ID
username      — 표시 이름
is_public     — 공개 여부 (boolean, 기본 false)
access_token  — Spotify 액세스 토큰 (암호화)
refresh_token — Spotify 리프레시 토큰 (암호화)
token_expiry  — 토큰 만료 시각
created_at    — 가입일
```

### albums 테이블
```
id              — 고유 ID
user_id         — users.id 참조
spotify_album_id — Spotify 앨범 ID
title           — 앨범 제목
artist          — 아티스트
genre           — 장르 (자동 분류)
year            — 발매 연도
cover_url       — 커버 이미지 URL
added_at        — 보관함에 추가한 날짜
```

## 5. 화면 구성

### 메인 (로그인 전)
- 사이트 소개 + Spotify 로그인 버튼
- 공개 유저 보관함 예시 미리보기

### 대시보드 (로그인 후)
- 상단: 총 앨범 수, 아티스트 수, 총 재생시간
- 중앙: 장르 비율 차트 + 연도별 분포 차트
- 하단: 최근 저장 앨범 리스트

### 보관함 페이지
- 기존 music-archive 장르 탐색 구조 그대로
- 좌측 필터 + 우측 앨범 리스트

### 앨범 상세 페이지
- 기존 music-archive 앨범 상세 디자인 그대로

### 공개 프로필 페이지 (`/u/username`)
- 해당 유저의 대시보드를 읽기 전용으로 표시

## 6. 구현 단계

### 1단계 — 인증 구축
- Supabase 프로젝트 생성
- Spotify OAuth 연동 (Netlify Function으로 토큰 교환)
- 로그인 / 로그아웃 처리
- 유저 정보 Supabase 저장

### 2단계 — 보관함 동기화
- Spotify `/v1/me/albums` 전체 페이지네이션 처리
- 장르 자동 분류 로직
- Supabase albums 테이블에 저장

### 3단계 — 시각화 구현
- Chart.js 연동
- 장르 비율, 연도별 분포, 아티스트 순위 차트

### 4단계 — 탐색 및 상세 페이지
- 기존 music-archive 코드 재활용
- 필터링, 앨범 상세 페이지 연동

### 5단계 — 공개/비공개 설정
- 설정 페이지
- 공개 프로필 페이지 구현

---

## 7. 미결정 사항

- 장르 자동 분류 정확도 — Spotify 아티스트 장르 태그 기반으로 할지, 직접 매핑 테이블을 만들지
- 토큰 보안 처리 방식 — Supabase에 저장할지 클라이언트 localStorage에만 둘지
- 유저명 설정 방식 — Spotify 이름 그대로 쓸지, 별도 설정 가능하게 할지
